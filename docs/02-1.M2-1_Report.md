# 지금까지 정리

## M1에서 끝난 것

* 프로젝트/타겟/answer/approve/run_auto 기본 골격
* state merge
* validation / retry / replan 최소 루프
* skill registry
* `ops.onboard.two_node` 대표 skill 등록
* `run_stub`로 skill 선택 + 입력 검증 + precheck
* `execute_stub`로 precheck / probe / decision / report summary 생성
* summary artifact 저장
* generic project 초기 상태 정리

## M2-1에서 끝난 것

* `decision` 결과가 `approval_needed`로 이어짐
* `needs_approval / resolve` 상태 전이
* `approve_stub`로 승인 저장
* 승인 후 `planned / execute` 전이
* `install_subagent` job stub 실행
* `approved_install_plan` 생성
* summary에 `install_planned` 반영

## 지금 꼭 할 것

테스트용 강제 approval 코드는 원복해야 한다.

파일:

* `core/app/services/skill_runner.py`

함수:

* `_decide_subagent_install_for_target()`

원복:

```python
if health_ok:
    return {
        "target_id": target_id,
        "decision": "already_present",
        "needs_approval": False,
        "reason": "Subagent health endpoint already responds successfully.",
        "recommended_next_action": "Skip installation and continue onboarding.",
    }
```

재빌드:

```bash
docker compose build core
docker compose up -d core
```

# M2-2 목표

이제 `install_subagent`가 단순 계획 출력이 아니라, **실행 가능한 설치 초안 artifact**를 만들게 하자.

즉 이번 단계는:

* install job이 target별 bootstrap/install script 초안을 생성
* 그 결과를 artifact로 저장
* summary에 설치 계획 파일 정보 반영

아직 실제 설치 실행까지는 안 간다.
이번 단계는 **실행 가능한 초안 생성**까지다.

  넣기
