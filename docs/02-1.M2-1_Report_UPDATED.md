# M2-1 완료 보고 (최신 정리본)

## 1. 목표

M2-1의 목표는 `decision` 결과를 실제 승인 흐름과 연결하는 것이었다.

즉,

- probe / decision에서
- `approval_needed`가 나오면
- 사용자가 승인하고
- 그 승인 상태를 다시 실행 흐름에 반영

하는 구조를 만드는 단계였다.

---

## 2. 완료된 내용

- `decision` 결과가 `needs_approval / resolve` 상태로 전이됨
- `approve_stub` 또는 승인 저장 endpoint가 추가됨
- 승인 정보가 project state에 저장됨
- 승인 후 다시 실행 시 `install_subagent` 또는 다음 단계가 이어질 수 있는 구조가 생김
- summary에 approval 필요 여부가 반영됨

---

## 3. 의미

M2-1 이전에는 skill이 “보고만 하는” 구조에 가까웠다면, M2-1 이후에는 최소한 **운영에서 필수인 승인 게이트**가 들어갔다.

즉 ChassisClaw는 이 시점부터,

- 자동 판단
- human approval
- 이후 실행 재개

를 다룰 수 있는 구조가 되었다.

---

## 4. 한계

- 승인 후 install은 아직 stub 수준
- 실제 운영형 bootstrap까지는 아직 도달하지 않음
- target identity 문제는 이후 단계에서 더 심화됨

---

## 5. 새 채팅 해석 기준

이 문서는 단독 기준 문서가 아니라, M2 전체 흐름의 첫 조각으로 보면 된다.  
새 채팅에서는 M2-1을 별도 목표로 다시 구현하기보다, **이미 완료된 것으로 보고 M3 자산 중심 리팩토링으로 넘어가면 된다.**
